# 娘家保健生技測試網站

模擬娘家保健生技官方網站的靜態測試網站，用於 UXAgent 自動化測試。

## 快速啟動

```bash
python3 -m http.server 3000
```

然後在瀏覽器開啟：http://localhost:3000

## 專案結構

```
├── index.html           # 商品列表頁（首頁）
├── product-detail.html  # 商品詳情頁
├── cart.html            # 購物車頁面
├── css/style.css        # 樣式
├── js/                  # JavaScript
└── images/              # 圖片資源
```

## 內建 UX 痛點

此網站故意埋入了 **3 個** UX 痛點，用於測試 UXAgent 的問題偵測能力。

---

### 1. 加入購物車完全無反饋
- **位置**: `js/product-detail.js:167-169`, `js/products.js:94-97`
- **問題**:
  - 點擊「加入購物車」後沒有任何視覺反饋
  - 按鈕沒有 loading 狀態或動畫
  - 沒有 toast 通知或提示訊息
  - 只有購物車徽章數字會變化（但不明顯）
- **影響**: 使用者完全不確定操作是否成功，可能重複點擊或放棄購買
- **分類**: ⚡ 系統響應與互動效能

---

### 2. 折扣價和原價難以區分
- **位置**: `css/style.css:547-555`
- **問題**:
  - 原價字體 18px，折扣價字體 19px，大小極為接近
  - 兩者都使用淡灰色系（#bbbbbb, #999999）
  - 使用者很難快速識別哪個是原價、哪個是折扣價
- **影響**: 無法有效傳達優惠訊息，使用者難以判斷划算程度
- **分類**: 🎨 視覺色彩與感知體驗

---

### 3. 數量調整按鈕和文字過小難以操作
- **位置**: `css/style.css:653-675`
- **問題**:
  - 加減數量按鈕只有 18x18px，遠低於建議的 44x44px 點擊目標
  - 數量輸入框只有 35x18px，過於狹小
  - 按鈕字體只有 10px，且顏色淡（#999999）
  - 數字輸入框字體只有 9px，難以辨識
  - 「數量」說明文字也偏小
- **影響**: 點擊目標過小容易誤觸，文字太小難以閱讀
- **分類**: 📐 導航、佈局與操作簡潔性

---

## 測試 UXAgent

```bash
# 啟動網站
python3 -m http.server 3000

# 執行 UXAgent（另開終端機）
cd /Users/akiraeason/Desktop/UXAgent
uv run -m src.simulated_web_agent.main \
  --intent "瀏覽商品並加入購物車" \
  --start-url "http://localhost:3000" \
  --max-steps 20
```

## 痛點分類說明

| 分類 | 痛點編號 | 說明 |
|------|---------|------|
| ⚡ 系統響應與互動效能 | #1 | 加入購物車無反饋 |
| 🎨 視覺色彩與感知體驗 | #2 | 折扣價和原價難以區分 |
| 📐 導航、佈局與操作簡潔性 | #3 | 數量按鈕過小難以操作 |

**共 3 個痛點**，每個都經過驗證且真實存在於程式碼中，確保測試網站能夠有效評估 UXAgent 的問題偵測能力。
